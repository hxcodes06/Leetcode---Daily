# Day 5 - Project Employees I

### LeetCode Problem

**Problem Number:** 1075

**Difficulty:** Easy

---

## Problem Statement

You are given two tables: `Project` and `Employee`.

- The `Project` table stores the projects and the employees working on them.
- The `Employee` table stores each employee's name and years of experience.

The task is to calculate the **average years of experience** for employees working on each project and round the result to **2 decimal places**.

Return the result in any order.

---

## Example

### Input

**Project**

| project_id | employee_id |
|------------|-------------|
| 1 | 1 |
| 1 | 2 |
| 1 | 3 |
| 2 | 1 |
| 2 | 4 |

**Employee**

| employee_id | name | experience_years |
|-------------|------|------------------|
| 1 | Khaled | 3 |
| 2 | Ali | 2 |
| 3 | John | 1 |
| 4 | Doe | 2 |

### Output

| project_id | average_years |
|------------|---------------|
| 1 | 2.00 |
| 2 | 2.50 |

---

## My Approach

Since the experience of each employee is stored in a different table, I first joined the `Project` and `Employee` tables using the `employee_id` column.

After combining the required data, I grouped the records by `project_id` so that each project's employees were considered together. Finally, I calculated the average experience using the `AVG()` function and rounded the result to two decimal places using `ROUND()`.

---

## Strategy

- Join the `Project` and `Employee` tables using `employee_id`.
- Group the records by `project_id`.
- Calculate the average experience for each project.
- Round the average to two decimal places.
- Return the project ID and average experience.

---

## Algorithm

1. Join the `Project` and `Employee` tables on `employee_id`.
2. Group the records by `project_id`.
3. Calculate the average of `experience_years` using `AVG()`.
4. Round the result using `ROUND(..., 2)`.
5. Display the result as `average_years`.

---

## SQL Solution

```sql
SELECT
    project_id,
    ROUND(AVG(experience_years), 2) AS average_years
FROM Project p
JOIN Employee e
ON p.employee_id = e.employee_id
GROUP BY project_id;
```

---

## Time Complexity

**O(n)**

The query performs one join and one grouping operation over the records.

---

## Space Complexity

**O(1)**

No additional space is used apart from the result generated by the query.
